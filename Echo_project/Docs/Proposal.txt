#### 1. Executive Summary
This proposal outlines the development of **Echo Custom**, a specialized local command-execution chatbot designed to enhance Eric Wilson's red team operations, penetration testing engagements, and general system administration tasks on Linux/Windows environments. Echo operates in "shell-only mode" with no safety restrictions, allowing users to request and execute arbitrary shell commands via an HTTP API (e.g., running on localhost). The bot maintains context across sessions using persistent files, logs interactions for auditing purposes, summarizes long conversations to control token usage, and enforces structured tool calls.

Echo is built as a Python script (Echo_custom.py) with key features including:
- **System Setup**: Defines ANSI colors for console output, file paths for context and log storage, API URL, and an empty system prompt that can be filled dynamically or via user input.
- **Context Management**: Loads previous session context from `Echo-context.txt` if available; otherwise starts fresh. This ensures continuity across multiple runs without losing important findings.
- **Logging**: Records user messages, assistant responses, and tool outputs in JSONL format for replayability and auditing.
- **Token Estimation & Summarization**: Automatically summarizes long conversation histories to prevent exceeding token limits while preserving key information (e.g., summarizing earlier parts when the context reaches a certain size).
- **Main Loop**: Accepts user input, sends requests to the API, parses responses for command execution (via regex), executes commands securely using `subprocess`, captures output, and continues reasoning or exits based on keywords.

The build leverages Eric's existing infrastructure (e.g., running on localhost) and focuses on efficiency, reliability, and red team-specific behaviors (e.g., permission requests before execution, structured tool calls).
